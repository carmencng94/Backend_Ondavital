## Why

Currently, two major issues affect the Google Calendar synchronization in the room reservation system:
1. Confirmed bookings block all rooms instead of only the specific room that was reserved. This happens because the availability query uses Google's `freebusy.query` API, which only returns blind "busy" time ranges without room-specific context, causing the system to block that time slot globally for all rooms.
2. Synchronized events appear faded, hatched, or gray/translucent in the administrator's Google Calendar. This occurs because the events are inserted by a service account, leaving the calendar owner (David) with a pending attendee response status (`needsAction`).

## What Changes

- **Room-Specific Google Calendar Blocking**: Replace the blanket time blocking from Google Calendar with a filtered approach. The system will retrieve calendar events with metadata (via `calendar.events.list`) and ignore events generated by Onda Vital. The local SQLite database will handle the room-specific blocking for Onda Vital's own bookings, while only David's personal calendar events will block all rooms.
- **Auto-Accepted Calendar Events**: Automatically set the attendee status of the calendar owner to `'accepted'` and suppress email invitations (`sendUpdates: 'none'`) when inserting or updating events from the backend, so that events display in full color (confirmed) immediately.
- **De-duplication in Global Calendar**: Filter out Onda Vital's synchronized events when retrieving the day's global occupancy list to prevent duplicates from showing up under "Calendario de David".
